# Документация проекта Diploma_MTC

## Описание базового синтаксиса записи и форматирования

**Финальный проект по ручному тестированию**: https://zubinaad-qa120-2.yonote.ru/share/52e931bf-5946-4723-8fb2-72cc201ea285

 **Репозиторий проекта по автотестированию**: https://github.com/elen-zd/Diploma_MTC.git

Проект содержит набор автоматизированных тестов для тестирования интернет-магазина МТС (shop.mts.ru):

- **UI-тесты** - проверка функциональности (поиска товаров, добавления в корзину, навигации по категориям)
- **API-тесты** - проверка операций с корзиной через REST API (добавление, получение, редактирование, удаление товаров)

Проект использует следующие технологии и синтаксис:

- **Python** - Основной язык программирования для написания тестов.
- **Selenium** - Библиотека для автоматизации взаимодействия с веб-браузером (UI-тесты).
- **Requests** - Библиотека для выполнения HTTP-запросов к API.
- **Pytest** - Фреймворк для написания и запуска тестов с использованием маркеров (@pytest.mark.ui, @pytest.mark.api).
- **Allure** - Инструмент для генерации детальных отчетов о выполнении тестов с пошаговой детализацией.
- **ChromeDriver** - драйвер браузера Chrome для Selenium.

### Структура проекта
- **Папка test_mts** - папка проекта, содержит все файлы:
тесты, конфигурацию, фикстуры, объекты страниц и документацию 
  - **Папка pages_shop** - содержит классы для работы с UI и API по паттерну Page Object
      - Файл ApiMTC.py - содержит класс CartAPIClient с методами для выполнения HTTP-запросов
      - Файл ShopMTC.py - содержит класс Shop с методами действий в браузере
  - **Файл config.py** - содержит настройки проекта:
базовые URL, таймауты, селекторы элементов, тестовые данные
  - **Файл conftest.py** - содержит фикстуры pytest:
driver (инициализация и закрытие Chrome),
cart_client (создание API-клиента и очистка корзины)
  - **Файл test_api.py** - содержит тесты для API проверок 
  - **Файл test_ui.py** - содержит тесты для UI проверок
  - **Файл pytest.ini** - содержит маркеры для запуска тестов
  - **Файл README.md** - содержит документацию:
описание проекта, структура, инструкции по запуску
  - **Файл requirements.txt** - содержит список зависимостей проекта:
pytest, selenium, requests, allure-pytest
  - **Файл run.sh** - содержит скрипт для:
запуска тестов с генерацией и открытием Allure-отчета


### Форматирование кода
- Код форматируется в соответствии с PEP 8 (стиль написания кода на Python).
- Методы и функции тестов задокументированы.
- Все шаги теста размечаются с помощью `with allure.step` для улучшения читаемости отчетов.
- Использован паттерн Page Object для UI-тестов (класс Shop в ShopMTC.py).
- Использован паттерн API Client для API-тестов (класс CartAPIClient в ApiMTC.py).
- Все конфигурационные данные вынесены в файл config.py (URL, селекторы, тестовые данные).

## Инструкция по запуску тестов для формирования отчета Allure

### Предварительная настройка
- Установите Python 3.8 или выше.

- Установите необходимые зависимости:
```bash
  pip install -r requirements.txt
```
Или установка вручную:
```bash
  pip install pytest==9.0.2 selenium==4.41.0 requests==2.32.5 allure-pytest==2.15.3 webdriver-manager==4.0.2
```

- Убедитесь, что у вас установлен Google Chrome
- ChromeDriver установится автоматически при первом запуске тестов благодаря webdriver-manager.


### Запуск тестов

- Запуск всех тестов:
```bash
    pytest
```
- Запуск всех тестов с подробным выводом в консоль:
```bash
    pytest -v
```
- Запуск только UI-тестов:
```bash
    pytest -m "ui" -v
```
- Запуск только API-тестов:
```bash
    pytest -m "api" -v
```

## Ручной запуск тестов и формирование Allure отчета

### Пошаговая генерация отчета

1. Запуск тестов и сохранение результатов:

Эта команда запускает тесты и формирует папку `results` с результатами:
```bash
    pytest --alluredir=./results
```
2. Генерация HTML-отчета:
Эта команда генерирует отчет на основе результатов тестов:
```bash
    allure generate ./results -o ./final-report --clean
```
3. Просмотр отчета в браузере:
Эта команда открывает сгенерированный отчет в браузере по умолчанию:
```bash
    allure open ./final-report
```
### Быстрый запуск с помощью run.sh

* Для удобства работы с проектом создан скрипт в файле `run.sh`, 
который автоматизирует процесс запуска тестов и формирования Allure отчета.

* Что делает скрипт:
  - Очищает старые результаты - удаляет папку с предыдущими результатами тестов
  - Запускает тесты - выполняет pytest с сохранением Allure результатов в папку ./results
  - Сохраняет историю - переносит историю предыдущих отчетов для построения графиков трендов
  - Генерирует отчет - создает HTML отчет Allure в папке ./final-report
  - Открывает отчет - автоматически открывает сгенерированный отчет в браузере

1. Как использовать run.sh:
 - Выполнить команду в терминале:
   ```bash
   ./run.sh
   ```
